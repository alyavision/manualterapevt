# Инструкция по оптимизации изображений

## Проблема
Изображения имеют большой размер (3.5 МБ) и не оптимизированы для разных размеров экрана.

## Решение

### 1. Создание адаптивных версий изображений

Для каждого изображения нужно создать 3 версии:

#### Изображения в WhyChooseUsSection:
- `/3608565_1764144829327469900-ezgif.com-video-to-webp-converter.webp` (925.7 KiB)
- `/3609723_1764146891488837400-ezgif.com-video-to-webp-converter.webp` (512.3 KiB)
- `/3609734_1764146834925700400-ezgif.com-video-to-webp-converter.webp` (423.7 KiB)
- `/3609894_1764147219495721100-ezgif.com-video-to-webp-converter.webp` (774.7 KiB)

**Создать версии:**
- `-380w.webp` - для мобильных (380px ширина)
- `-520w.webp` - для планшетов (520px ширина)
- `-800w.webp` - для десктопов (800px ширина, можно оставить оригинал)

#### Изображение в AboutUsSection:
- `/3606768_1764141440762261800-ezgif.com-video-to-webp-converter.webp` (824.4 KiB)

**Создать версии:**
- `-380w.webp` - для мобильных (380px)
- `-420w.webp` - для планшетов (420px)
- `-640w.webp` - для десктопов (640px)

#### Изображение в HeaderSection:
- `/Ruslan%20Artjkov.webp` (62.6 KiB)

**Создать версии:**
- `-348w.webp` - для мобильных (348px)
- `-520w.webp` - для планшетов (520px)
- `-640w.webp` - для десктопов (640px)

### 2. Инструменты для оптимизации

#### Онлайн инструменты:
1. **Squoosh** (https://squoosh.app/) - лучший инструмент для сжатия WebP
2. **TinyPNG** (https://tinypng.com/) - сжатие изображений
3. **ImageOptim** - для Mac

#### Командная строка:
```bash
# Установка sharp-cli
npm install -g sharp-cli

# Оптимизация одного изображения
sharp -i input.webp -o output-380w.webp --resize 380

# Пакетная обработка
for file in *.webp; do
  sharp -i "$file" -o "${file%.webp}-380w.webp" --resize 380
  sharp -i "$file" -o "${file%.webp}-520w.webp" --resize 520
done
```

### 3. Настройки качества

Рекомендуемые настройки для WebP:
- **Качество:** 75-85% (баланс между размером и качеством)
- **Метод сжатия:** 6 (хороший баланс скорости/качества)
- **Lossless:** false (для фотографий)

### 4. После создания версий

Обновить компонент `ResponsiveImage.tsx` для использования srcset:

```typescript
const generateSrcSet = (baseSrc: string, widths: number[]): string => {
  return widths
    .map((width) => {
      const baseName = baseSrc.replace(/\.[^.]+$/, "");
      const extension = baseSrc.match(/\.[^.]+$/)?.[0] || ".webp";
      return `${baseName}-${width}w${extension} ${width}w`;
    })
    .join(", ");
};
```

### 5. Ожидаемый результат

После оптимизации:
- **Текущий размер:** 3,523.5 KiB
- **Ожидаемый размер:** ~85-105 KiB (экономия ~3,438 KiB)
- **Улучшение LCP:** Значительное улучшение времени загрузки

### 6. Проверка

После оптимизации проверить:
1. Размер файлов уменьшился
2. Качество изображений приемлемое
3. Страница загружается быстрее
4. LCP улучшился
